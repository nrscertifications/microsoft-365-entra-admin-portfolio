# Group-Based Collaboration Management

## Administrative Objective

Create, configure, and validate Microsoft 365 group administration across Microsoft 365 admin center and Microsoft Entra admin center.

This section demonstrates how group creation, ownership, membership, and Entra-side validation support collaboration and access-related troubleshooting.

---

## Work Completed

* Created a Microsoft 365 group from the Microsoft 365 admin center.
* Configured group properties and collaboration settings during the creation workflow.
* Added and validated group members.
* Added and validated group owners.
* Completed the group creation workflow.
* Validated the Microsoft 365 group object from the Microsoft Entra admin center.
* Checked Entra-side group properties and configuration views to understand where group objects are managed outside the Microsoft 365 admin center.

## Support Relevance

Groups are central to collaboration, access, ownership, and communication workflows in Microsoft 365.

Support teams need to understand group type, membership, ownership, and where group properties are checked when troubleshooting access, distribution, Teams/SharePoint collaboration, or ownership-related issues.

Validating the group from both Microsoft 365 admin center and Microsoft Entra admin center helps prevent confusion when a group is created in one portal but must be supported or investigated from another.

---

## Outcome

A Microsoft 365 group was created, configured, and validated through its properties, members, owners, completion state, and Entra-side object view.

This supports practical collaboration and access troubleshooting scenarios where group membership, ownership, and object visibility affect user access.
