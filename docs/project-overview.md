# Project Overview

This portfolio documents practical Microsoft 365 and Microsoft Entra ID administration work performed in a dedicated lab tenant using fictional users and test objects.

The repository is organized by operational workstream instead of course section. That structure makes the evidence easier for hiring managers, service desk leads, and technical reviewers to scan without needing context from the training material.

## Administrative Areas Covered

* Tenant foundation and custom domain readiness
* Microsoft 365, Azure, and Entra admin portal navigation
* Internal user provisioning and cross-portal validation
* Bulk user creation workflow
* External contacts and guest collaboration workflow
* Microsoft 365 group creation, owner assignment, and membership validation
* Licensing and service access review
* Service health, network insights, and backup readiness exposure
* PowerShell administration fundamentals staged as a separate follow-up track

## Positioning

This portfolio supports IT Support Technician, Service Desk Analyst, Help Desk Analyst, Technical Support Analyst, Desktop Support, and junior systems administration roles where Microsoft 365 and identity administration are part of the support environment.

The work is documented in a support-ready format: what was configured, why it matters, what was verified, and what evidence supports the result.
