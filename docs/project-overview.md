# Project Overview

This portfolio documents practical Microsoft 365, Microsoft Entra ID, Windows Server AD DS, and hybrid identity administration completed in dedicated non-production environments with fictional users and test objects.

The repository is organized by operational workstream rather than by the order in which the tasks were completed. That structure lets a hiring manager, service desk lead, or technical reviewer scan the summary first and then open the exact documentation and screenshots supporting each claim.

## Administrative Areas Covered

* Tenant foundation and custom domain readiness
* Microsoft 365, Azure, and Microsoft Entra administration portals
* Internal and bulk user provisioning with cross-portal verification
* External contacts and Microsoft Entra B2B guest workflows
* Microsoft 365 group ownership and membership administration
* Licensing and service access review
* Service health, network insights, Log Analytics exposure, and Microsoft 365 Backup readiness
* Microsoft Graph PowerShell user, group, license, bulk provisioning, and cleanup workflows
* Role assignment, administrative units, scoped delegation, Purview role groups, and PIM-style eligible access
* IDFix directory attribute remediation
* Windows Server AD DS and DNS deployment
* Microsoft Entra Connect Sync and Microsoft Entra Cloud Sync
* Synchronization scoping, user verification, Connect Health, and provisioning audit review

## Evidence Standard

Each detailed document states what was configured, what the screenshot proves, and where a result remained a review step or follow-up item. Fictional identities stay visible when they help the reviewer follow the workflow. Temporary passwords, public IP information, and values that could expose access are removed or cropped.

## Positioning

The portfolio supports IT Support Technician, Service Desk Analyst, Help Desk Analyst, Technical Support Analyst, Desktop Support, Junior Systems Administrator, and junior Microsoft 365 administration roles. It shows hands-on configuration and troubleshooting while keeping the non-production scope clear.
