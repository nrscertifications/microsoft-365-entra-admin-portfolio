# Screenshot Evidence

Screenshots are organized by administration workstream rather than by course section.

The screenshots use fictional lab users and test objects. Fake names remain visible when they make the workflow evidence understandable. Temporary passwords, secrets, public IP information, and access-enabling values are removed or cropped before publishing.

- `14-defender-security-reports-alerts` — 32 screenshots covering Microsoft Secure Score, recommended actions, Defender XDR incidents/reporting surfaces, Threat analytics, and Threat analytics email-notification configuration.
- `15-defender-email-collaboration-protection` — 69 screenshots covering Defender for Office 365 protection policies, Safe Attachments, Safe Links, allow/block controls, quarantine policy administration, Explorer remediation, Threat Tracker, and restricted-entity review.
